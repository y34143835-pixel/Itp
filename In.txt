These "Failed to resolve" errors happen because two things:

1.  Repository Missing: Some of these libraries (like RemotePreferences) live on
    a server called JitPack, which isn't in your current setup.
2.  Typos: Your log shows tv.danmaku.jjk (with two 'j's), but the library is
    actually tv.danmaku.ijk (with an 'i').

Let’s fix this. We need to update two files again. Please replace the code
exactly as shown below:

1. File: settings.gradle.kts (In the main root folder)

We need to tell Gradle to look at the JitPack server and the Xposed server.

pluginManagement {
    repositories {
        google()
        mavenCentral()
        gradlePluginPortal()
    }
}
dependencyResolutionManagement {
    repositoriesMode.set(RepositoriesMode.FAIL_ON_PROJECT_REPOS)
    repositories {
        google()
        mavenCentral()
        // Add these two lines carefully:
        maven { url = uri("https://jitpack.io") }
        maven { url = uri("https://api.xposed.info/") }
    }
}
